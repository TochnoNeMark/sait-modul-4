# OpenWing — учебный сайт авиакомпании (Модуль 4)

Многостраничный адаптивный сайт вымышленной авиакомпании OpenWing.
Чистые HTML + CSS + JS, без сборщиков, фреймворков и зависимостей.
Хостинг — статика (совместимо с GitHub Pages). Репозиторий: `TochnoNeMark/sait-modul-4`.

## Как со мной работать

- Отвечай и комментируй **на русском**. Код-комментарии — по-русски (в JS/CSS допустимо кратко).
- Все пользовательские тексты, плейсхолдеры, надписи на кнопках — **на русском**.
- Давай **production-ready** решения сразу, без промежуточных «упрощённых» вариантов.
- Предугадывай best practices и применяй по умолчанию, не жди уточняющих вопросов.
- **Перед каждым коммитом** показывай короткую сводку планируемых изменений и коммить только после подтверждения.
- Один коммит — одно логическое изменение. Сообщения коммитов — на русском, в описательном стиле.
- Шрифт всего сайта — строго **Montserrat Alternates** (для календаря исторически допускался обычный Montserrat). Poppins и любые другие шрифты — НЕ использовать.

## Запуск и проверка

- Локально: открыть любой `*.html` напрямую в браузере (`file://`) — всё работает без сервера.
- Перед тем как сообщить о готовности правки, **обязательно проверяй результат**: лови JS-ошибки (console/pageerror), проверяй верстку на десктопе (~1440), планшете (~820) и мобайле (~390), отсутствие горизонтального переполнения и функциональность (клики, формы, навигацию). Подойдёт headless-браузер (Playwright/Chromium) с измерением геометрии через DOM и скриншотами.
- WebP-ассеты оптимизируй (товарные фото — квадрат на белом фоне, баннеры — по ширине), вес держи в десятках КБ.

## Архитектура

### Страницы (каждый раздел — отдельная страница)
- `index.html` — главная (div 1): hero-баннер (→ `pricing.html`) + сетка из 8 городов.
- `tickets.html` — Билеты (div 1.2): форма с 5 состояниями (см. ниже).
- `events.html` — События (div 1.3): плакат + кнопка «Записаться» (оба → `enroll.html`).
- `reviews.html` — Отзывы (div 1.4): форма отзыва + бесконечная лента.
- `merch.html` — Мерч (div 1.5): сетка товаров с переключением категорий.
- `product.html?id=<id>` — карточка товара (div 1.5.1) + отзывы на товар (div 1.5.2). Один шаблон-генератор на все товары.
- `enroll.html` — форма записи на курсы пилотов (div 1.3.1).
- `cart.html` — корзина мерча.
- `about.html` — О нас (div 1.6): плакат слева + карточка с текстом о компании справа, кнопка «Смотреть тарифы» (→ `pricing.html`). Структурно — как `events.html`.
- `pricing.html` — Тарифы (div 1.6.1–1.6.3): промо OpenWing+ (подписка) + три раздела услуг (Места / Питание / Дополнительно) + отдельная корзина услуг + модалка «Подробнее». Своя шапка с якорной навигацией. Детали — в «Ключевых механиках».
- `404.html` — кастомная страница ошибки. Отдаётся GitHub Pages на несуществующих путях; также на неё ведёт клик по логотипу в шапке. Композиция «4 ⬭ 4», где «0» — иллюминатор с летящими облаками (строго ниже линии-шторки); нижняя подпись выбирается случайно при каждом заходе из трёх вариантов.

### Шапка
Синяя «пилюля»: **логотип-иллюминатор** (белый инлайн-SVG, ведёт на `404.html`), навигация (О нас/Билеты/События/Отзывы/Мерч), «Корзина», **белая иконка дома** (→ `index.html`; на главной — плавный скролл вверх), бургер на мобайле. На главную ведёт только иконка дома — пункта «Главная» в навигации нет. Поиска (лупы) НЕТ — намеренно убрана. Активный пункт навигации подсвечивается по текущей странице (в `main.js`, без скролл-спая). На странице товара активна «Мерч», на `enroll.html` — «События».

**Исключение — `pricing.html`:** своя шапка. Вместо обычной навигации — якорные кнопки **OpenWing+ / Места / Питание / Дополнительно** (скроллят к разделу страницы, активный подсвечивается скролл-спаем в `pricing.js`). «Корзина» в шапке показывает сумму корзины услуг и ведёт якорем к ней (`#svc-cart`), а не на `cart.html`.

### CSS
- Точка входа `styles/style.css` импортирует по порядку: `reset.css` → `fonts.css` → `main.css` → `adaptive.css`. **Порядок важен**: `adaptive.css` грузится последним, иначе media-оверрайды перебиваются базовыми правилами.
- `main.css` — токены (CSS-переменные) и все базовые стили (desktop-first).
- `adaptive.css` — только медиазапросы (планшет ≤900, мобайл ≤600/≤560). Сетки карточек (мерч и услуги) переключаются: ≤1000 → 2 колонки, ≤600 → 1 колонка.
- `fonts.css` — подключение шрифтов (Montserrat Alternates).
- Единые размеры шрифта через переменные: `--fs-h` (все заголовки), `--fs-p` (все параграфы), `--fs-btn` (текст всех кнопок). Между элементами могут отличаться только цвета, не размеры.
- Цветовые токены: `--c-header` #4d8aff, `--c-pill` #a6c4ff, `--c-home` #a5c3ff, `--c-accent` #4d89ff, `--c-logo` #ffffff (цвет логотипа-иллюминатора). Адаптив — на flex/grid и `clamp()`, без абсолютного позиционирования по пикселям.

### JS (модули по страницам, подключаются точечно)
- `catalog.js` — на всех страницах, **кроме `pricing.html`** (там своя логика услуг). Каталог `window.PRODUCTS`, категории `window.CATEGORIES`, цвета `window.COLORS`, корзина мерча `window.Cart` (localStorage). Обновляет текст кнопки «Корзина» в шапке.
- `main.js` — **на всех страницах**. Активный пункт навигации по текущей странице, бургер-меню, кнопка «домой», мост «город из главной → поле Куда» (через `sessionStorage`).
- `booking.js` — только `tickets.html`. Контроллер формы билетов.
- `reviews.js` — `reviews.html` и `product.html`. Переиспользуемый движок отзывов.
- `merch.js` — только `merch.html`. Рендер сетки товаров + переключение категорий.
- `product.js` — только `product.html`. Рендер карточки товара по `?id`.
- `cart.js` — только `cart.html`. Рендер корзины мерча.
- `pricing.js` — только `pricing.html`. Каталог услуг, отдельная корзина услуг, правила добавления, модалка «Подробнее», подписка OpenWing+, якорный скролл-спай. **`catalog.js` тут не подключается** — у услуг свой каталог и своя корзина.
- `enroll.js` — только `enroll.html`. Сабмит формы записи.
- `notfound.js` — только `404.html`. Случайный выбор нижней подписи при каждом заходе (без повтора предыдущей, через `sessionStorage`).

## Ключевые механики

**Корзина** (`catalog.js → window.Cart`): localStorage-ключ `openwing:cart`, объект позиций. Ключ позиции = `<id>__<colorKey>` (товар разного цвета = разные позиции). Цена/название берутся из каталога по id. Методы: `add(id, colorKey, qty)`, `setCount(key, n)`, `remove(key)`, `items()`, `total()`. Корзина общая для всех категорий. На странице корзины — степпер количества (−/+) и удаление, миниатюра тонируется в выбранный цвет. Кнопка «Корзина» в шапке показывает суммарную стоимость.

**Каталог**: у каждого товара `id`, `category` (`suitcase` / `pillow` / `electronics`), `name` («Чемодан 1», «Подушка 1», «Повербанк 1» …), `volume`/`volumeFull`, `price`, `rating`, `desc`, `img`. Дизайн карточки и страницы товара общий — между товарами меняются только название/описание/изображение/цена. Кнопка «Назад» на странице товара возвращает в `merch.html?cat=<категория>` (та категория, из которой открыли).

**Форма билетов** (`booking.js`): вкладки переключают содержимое панели:
- «Покупка» + тумблер **Билет/Отель** (Трансфера нет): перелёт (Откуда ⇄ Куда + даты) или отель (город из списка, счётчик гостей ±, заселение/выезд через календарь). Кнопка «Найти» сбрасывает форму.
- «Регистрация»: фамилия + номер заказа (ввод с клавиатуры), кнопка «Регистрация» в строке сбрасывает форму.
- «Бронирование»: фамилия + номер заказа + место (выпадающие Ряд 1-30 / Место A-F), кнопка «Забронировать место» снизу.
- «Статус рейса»: номер заказа, кнопка «Найти рейс» в строке.
- «Услуги»: ссылка на `pricing.html`.
Все кнопки подтверждения только сбрасывают форму — **ничего реально не отправляется и не оплачивается**. Город, выбранный на главной, подставляется в «Куда».

**Отзывы** (`reviews.js`): форма (имя + оценка звёздами + текст) добавляет отзыв наверх ленты; раз в 30 секунд генерируется случайный отзыв; на странице — бесконечная лента (подгрузка при прокрутке вниз). Отзывы **только в памяти**, стираются при перезагрузке (без localStorage). Набор текстов выбирается по `data-reviews="site|merch"`.

**Формы записи/билетов** интерактивны, но нефункциональны (демо): валидация минимальная, отправки на бэкенд нет.

**Тарифы и OpenWing+** (`pricing.js`, страница `pricing.html`):
- **Каталог услуг** — отдельный от мерча, внутри `pricing.js`. У услуги: `id`, `section` (`place`/`food`/`extra`), `name`, `price`, `img`, короткое и полное описание, флаги `single`/`max`/`plus`/`free`/`download`. Карточки услуг визуально как карточки мерча.
- **Отдельная корзина услуг** — localStorage-ключ `openwing:services` (не пересекается с корзиной мерча `openwing:cart`). Панель корзины услуг — внизу страницы, со степпером (−/+) и удалением.
- **Правила добавления:** *Места* и *Питание* — single-select (добавление услуги заменяет предыдущую из того же раздела, максимум 1). *Дополнительно* — услуги не заменяют друг друга, свои лимиты: трансфер ×1, лаундж ×1, доп. багаж ×2.
- **«Подробнее»** открывает модалку (как карточка товара, но без выбора цвета и без отзывов).
- **OpenWing+** — годовая подписка, демо-флаг localStorage `openwing:plus` (ничего реально не оплачивается), активирует кнопка «Быть в плюсе». После активации: цены всех услуг −5%; в «Питании» появляется бесплатный кофе (не заменяет еду); в «Дополнительно» — бесплатный журнал, при добавлении которого скачивается PDF. Отключение подписки убирает plus-услуги из корзины.
- **PDF-журнал** — `assets/openwing-plus-magazine.pdf`, собран из JPG через `img2pdf` (без перекодирования), качается по добавлению бесплатного журнала. Картинки услуг — в `assets/images/div6/services/*.webp`.

## Конвенции и важные нюансы

- Никаких внешних JS-библиотек и CSS-фреймворков. Только ванильный стек.
- Отзывы — намеренно эфемерны (без `localStorage`/`sessionStorage`). Корзина — наоборот, в `localStorage`.
- Иконки — инлайн-SVG (лупа удалена). Логотип-иллюминатор и облака (на `404.html`), белый домик, swap-стрелки.
- Изображения новых товаров/секций класть в `assets/images/div<N>/`, оптимизировать в WebP.
- В `assets/images/` лежат неиспользуемые ассеты прошлых модулей (chemodan.png, штампы, открытки и т.п.) — их не трогаем.
- Макет — в Figma: **https://www.figma.com/design/gtId93OdQxX9dXBkSth9wQ/Сайт-модуль-4** (ключ файла `gtId93OdQxX9dXBkSth9wQ`). Если правка идёт «по макету», ориентируйся на соответствующий фрейм (`div 1.x`, `404`, `Календарь`); точные размеры/цвета бери из макета, но единые размеры шрифта (см. выше) имеют приоритет над разнобоем в макете. Как достать данные из Figma — см. раздел «Figma API» ниже.
- Разделы **О нас и тарифы** (`div 1.6`, `div 1.6.1`, `div 1.6.1,5`, `div 1.6.2`, `div 1.6.3`) лежат в отдельном, более новом файле макета: ключ `lLCN7SjrYTX8JcmWKXvZCU`. Токен к нему Артур даёт в сессии.
- Адаптив проверяй на трёх ширинах: ~1440 (десктоп), ~820 (планшет), ~390 (мобайл).

## Figma API (как вытаскивать данные — инструкция для меня)

Токен Figma — **секрет, в репозиторий не коммитится**. Артур даёт personal access token (`figd_…`) в сессии; передавать в заголовке `X-Figma-Token`. Все запросы — к `https://api.figma.com/v1/`.

- **Ключ файла** и **node-id** берутся из URL макета: `figma.com/design/<КЛЮЧ>/...?node-id=<N-M>`. В URL node-id через дефис (`225-2`), в API — через двоеточие (`225:2`).
- **Один узел/фрейм** (с детьми): `GET /v1/files/<КЛЮЧ>/nodes?ids=<NODE>` → `nodes[NODE].document` (дерево слоёв).
- **Структура файла** (найти фрейм по имени): `GET /v1/files/<КЛЮЧ>?depth=2`. Фреймы названы `div 1.x`, страница ошибки — фрейм **`404`** на холсте `Сайт`.
- **Рендер фрейма в PNG** (визуальная сверка макета): `GET /v1/images/<КЛЮЧ>?ids=<NODE>&format=png&scale=2` → вернёт временный URL картинки, его и качай.
- **Цвета**: у узла массив `fills`; для `type:"SOLID"` цвет в `color {r,g,b,a}` (числа 0..1), hex = `round(канал*255)`. Фон фрейма дублируется в `backgroundColor`. Если `opacity`/`a` < 1 — учитывай прозрачность.
- Ответ большой — парсь питоном (`curl … -o /tmp/fig.json`, затем `json` + рекурсивный обход), а не глазами.

Маппинг цветов макета на токены сайта (**используй токены, не хардкодь hex**): `#4d8aff` = `--c-header`, `#a6c4ff` ≈ `--c-pill`, `#a5c3ff` = `--c-home`, `#4d89ff` = `--c-accent`, `#ffffff` = `--c-white`. Например, фон фрейма `404` в макете — `#a5c3ff` (= `--c-home`).
